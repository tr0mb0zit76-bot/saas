# Domain modules

## Platform modules

| Module | Purpose |
| --- | --- |
| Tenants | Company registration, status, settings |
| Billing | Plans, subscriptions, invoices, usage |
| Platform users | SaaS owner, admin, support access |
| Observability | Health, logs, metrics, incidents |
| Feature flags | Controlled rollout by tenant or plan |

## Tenant modules

| Module | Purpose |
| --- | --- |
| Users and roles | Tenant membership and permissions |
| Customers | Shippers, consignees, customers, contacts |
| Orders | Core transportation requests and workflow |
| Carriers | Carrier companies, vehicles, drivers |
| Documents | Templates, generated documents, files |
| Finance | Invoices, payments, margin, debt |
| Reports | Operational and financial analytics |
| Settings | Tenant dictionaries, statuses, legal details |
| Integrations | Email, maps, EDI, accounting, freight exchanges |

## External portal candidates

These are not MVP by default, but should be considered later:

- customer portal;
- carrier portal;
- public quote/request form;
- API clients.
