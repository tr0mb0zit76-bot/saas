# Справочник плейсхолдеров печатных форм (DOCX)

Канонический источник в коде: `PrintFormVariableCatalog`, `PrintFormPlaceholderPathResolver`, сервис `PrintFormPlaceholderGuide`.

В CRM арендатор открывает **Настройки → Шаблоны → «Справочник плейсхолдеров»** — тот же список, что описан здесь, в интерактивном виде с поиском.

## Сценарий арендатора

1. Компания арендует среду Traklo Pro (tenant).
2. Готовит свой DOCX-шаблон договора-заявки, счёта, поручения и т.д.
3. В Word вставляет макросы `${...}` по справочнику (короткие имена или полные пути).
4. Загружает файл в **Настройки → Шаблоны**.
5. На вкладке **Плейсхолдеры** проверяет сопоставление «макрос → поле CRM» и при необходимости меняет источник.
6. Генерирует тестовый документ по заказу или лиду.

## Два способа именования

| Способ | Пример в DOCX | Когда использовать |
| --- | --- | --- |
| **Короткое имя (legacy)** | `${inn}`, `${nomer_zayavki}` | Совместимость с макетами v5, привычные имена |
| **Полный путь данных** | `${customer.inn}`, `${order.order_number}` | Явное указание источника, новые шаблоны |

Префиксы сторон в ВЭД-заявках:

- `cp_` — заказчик (customer)
- `dp_` — перевозчик (carrier)
- `lp_` — своя компания (own_company)

## Группы полей (полные пути)

- **order.*** — номер, даты, ставки, оплата, СВХ, таможня
- **customer.*** / **carrier.*** / **own_company.*** — реквизиты, банк, подписант, ЭДО
- **route.*** — адреса, города, время погрузки/выгрузки
- **cargo.*** — сводка по грузу; **cargo_row_*** — таблица позиций (cloneRow)
- **driver.*** / **vehicle.*** — водитель и ТС
- **lead.*** / **offer.*** — лиды и коммерческие предложения

## Специальные макросы

| Макрос | Назначение |
| --- | --- |
| `${cargo_row_name}` | Таблица грузов (размножение строк) |
| `${route_point_row_address}` | Таблица точек маршрута |
| `${cp_basic_terms_row_text}` | Базовые условия заказчика |
| `${dp_basic_terms_row_text}` | Базовые условия перевозчика |
| `${signature}` / `${stamp}` | PNG подписи и печати |
| `${document_verification_qr}` | QR проверки документа |

## Где хранится сопоставление

- Глобальный каталог полей — PHP (`PrintFormVariableCatalog`)
- Автоматические короткие имена — `PrintFormPlaceholderPathResolver`
- Переопределения арендатора — `print_form_templates.settings.variable_mapping` (JSON)

## Связь с тарифами

Модуль **print** (печать DOCX/PDF) включается тарифом Pro+ или вручную суперадмином платформы (**Арендаторы → Модули** или **Тарифы → Модули**).
